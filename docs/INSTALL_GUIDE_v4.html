SDAD v4.3 — Installation Guide

How to install the methodology, verify prerequisites, and initialize your first project.

What the installer does

The SDAD installer runs 7 steps inside the project repo where you want the methodology active. It checks prerequisites, builds the .claude/ folder, downloads skill files, installs CLAUDE.md, initializes SPEC.md and LESSON_LIBRARY.md, and registers the Pyplan MCP server globally in Claude Code.

  1. Check prerequisites

    Verifies Node.js v18+, Claude Code, and git. Installs Claude Code automatically if missing.

  2. Create .claude/ folder structure

    Creates all required skill and hook directories under .claude/, including pyplan/mcp/.

  3. Download skill files

    Fetches all SKILL.md files from the SDAD repo into .claude/skills/, including the new pyplan-mcp skill.

  4. Install CLAUDE.md

    Downloads the core instruction file to the repo root. If a CLAUDE.md already exists, the SDAD block is appended — it is never overwritten.

  5. Initialize project files

    Creates SPEC.md (blank template) and LESSON_LIBRARY.md if they don't exist. Updates .gitignore with SDAD entries.

  6. Register Pyplan MCP server globally

    Runs claude mcp add pyplan https://dev.pyplan.com/ai/mcp --transport http to register the Pyplan MCP server in Claude Code. Skipped automatically if already registered. On first use, Claude Code will open the browser for Pyplan OAuth login. If the registration fails, the installer prints the manual command to run.

  7. Summary

    Prints installed files and the exact next step to start working.

Prerequisites

Requirement Minimum version Notes
Node.js v18 or higher Download from nodejs.org. The installer checks and will stop if missing.
Claude Code Latest Installed automatically by the installer if not found (npm install -g @anthropic-ai/claude-code).
git Any recent version Run the installer from inside a git repo. If no repo is detected, it runs git init automatically.
Internet connection — Required to download skill files from GitHub during installation.
Run the installer from the correct folder Navigate to the project repo root before running. SDAD installs into the current working directory.

Run the installer

Open a terminal, navigate to your project folder, and run:

bash <(curl -fsSL https://raw.githubusercontent.com/diegomondrik/sdad-v4/main/install.sh)

The script downloads and executes in a single step. No file is saved to your repo.

Option A — paste directly (recommended)

Open PowerShell, navigate to your project folder, and run:

$install = (Invoke-WebRequest -Uri "https://raw.githubusercontent.com/diegomondrik/sdad-v4/main/install.ps1" -UseBasicParsing).Content
Invoke-Expression $install

Option B — download first, then run

If you prefer to inspect the script before executing:

Invoke-WebRequest -Uri "https://raw.githubusercontent.com/diegomondrik/sdad-v4/main/install.ps1" -OutFile "install-sdad.ps1"
powershell -ExecutionPolicy Bypass -File ".\install-sdad.ps1"

Initialize a project

After the methodology installer runs once, use the project initializer at the start of each new project. It collects project name, developer name, client name, and compliance tier, then writes all scaffolding files. 

bash <(curl -fsSL https://raw.githubusercontent.com/diegomondrik/sdad-v4/main/project-init.sh)
$init = (Invoke-WebRequest -Uri "https://raw.githubusercontent.com/diegomondrik/sdad-v4/main/project-init.ps1" -UseBasicParsing).Content
Invoke-Expression $init

The initializer creates:

File / FolderPurpose
SPEC.mdPre-filled with project name, developer, date, and compliance tier
LESSON_LIBRARY.mdBlank lesson library for this project
DECISIONS.mdDesign decisions log — appended automatically by $build
.sdad/project.mdProject registry: name, developer, client, tier, session log
.sdad/flows/Empty directory for saved $flow definitions

Context budget monitor

Before every Claude Code session, run cc-status-line in a separate terminal window. It displays the current model, context usage percentage, session cost, and git branch in real time. SDAD uses context percentages at 50% and 65% as thresholds for session warnings. 

npx ccstatusline@latest
Keep it open in a separate terminal Run ccstatusline in one terminal tab, then start claude in a second tab. The status line updates automatically as you work.

Verify the installation

Start Claude Code (claude) and run these commands to confirm everything is active:

CommandExpected result
$sdadMethodology overview: all phases and commands listed
$skillsAI Architect and AI Engineer shown as always active
$specFirst requirements question with a proposed default
$pauseSession state including context budget percentage
Installation successful when all four commands respond correctly If any command is not recognized, verify that CLAUDE.md is present in the repo root and that the .claude/skills/ folder contains the SKILL.md files.

Troubleshooting

SymptomLikely causeFix
Installer stops at Node.js check Node.js not installed or version below 18 Install Node.js v18+ from nodejs.org and re-run
Skill file download fails No internet access or GitHub unreachable Check connection; download files manually from the repo and place them in .claude/skills/
$sdad not recognized in Claude Code CLAUDE.md missing from repo root Re-run the installer; check that the file landed at the repo root, not in a subfolder
PowerShell execution policy error Script execution blocked by system policy Use Option A (paste + Invoke-Expression) instead of running the .ps1 file directly
SPEC.md not created File already exists in the repo This is intentional — existing SPEC.md files are never overwritten
