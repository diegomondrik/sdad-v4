The five-phase loop

01
$spec
Context
SDAD reads your repo, asks one targeted question at a time, proposes sensible defaults. Sets the project language, domain, and compliance tier.
Requirements locked
02
$specout
Spec
Generates a full 13-section SPEC.md: vision, flows, data model, architecture, business rules, security, and definition of done.
SPEC.md written
03
$build
Build
One vertical increment at a time. Announces what it will touch, runs real tests after writing, records every decision.
Tests + DECISIONS.md
04
$qa
QA
Four mandatory layers: Security (P0/P1/P2) · Structure · Efficiency · Best Practices. Platform layer on Pyplan and Board projects.
QA report
05
$lesson / $pause
Learn
Patterns that can recur become Lesson Library entries. Mechanically verifiable ones become ratchet checks that run on every future commit.

phase 2

Quality review: auto (last increment) · review (manual) · full (whole project) · $QA standalone.

Platform Modes

Pyplan and Board projects

Set a single line in CLAUDE.md to activate platform-specific spec sections, increment checklists, QA layers, and skills.

PROJECT_PLATFORM: pyplan

Pyplan — Analytics & Planning

For influence-diagram models with nodes, interfaces, and optional MCP tool exposure. Adds data architecture gates, model snapshot versioning, and the $audit lifecycle.

  • §A (Data Architecture) — build gate
  • §D (MCP Tools Catalog) — build gate when @mcp_tool used
  • §B Discovery Log — tracks data deltas during build
  • QA Layer 5 — node graph, indexes, HTML interface, MCP surface
  • $audit — five-dimension audit of existing models
  • Model snapshots → .sdad/pyplan-snapshots/
Activate in CLAUDE.md
PROJECT_PLATFORM: pyplan
PROJECT_PLATFORM: board

Board — BI & Planning

For Board BI projects with entities, relationships, cubes, capsules, and procedures. Enforces entity creation order and algorithm syntax at every increment close.

  • §E (Board Data Model) — build gate
  • §F (Board Capsule Structure) — informational
  • BR-05 — entity creation order enforced (Entities → Relationships → Cubes)
  • BR-03 — algorithm syntax: block letters + Board functions only
  • BR-04 — scheduleable Procedures at Data Model level only
  • QA Layer 5 — 10 named checks (DM-01..06, CP-01..05, SEC-01)
Activate in CLAUDE.md
PROJECT_PLATFORM: board
